# Regras de Git — obrigatórias

Após qualquer modificação de código:

1. **Se a modificação pertence ao contexto da branch atual**: commite e faça push para a branch atual.
2. **Se a modificação está fora do contexto da branch atual**: crie uma nova branch com nome descritivo (`feat/`, `fix/`, etc.), commite e faça push para essa nova branch.
3. **Nunca faça push direto para `main`**, em nenhuma circunstância.
